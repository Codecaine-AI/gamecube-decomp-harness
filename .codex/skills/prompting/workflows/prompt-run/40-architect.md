# Architect

Choose the artifact architecture before drafting.

## Steps

1. Read `../../00-routing.md` to choose system prompt, input formatter, or both.
2. For system prompts, read `../../10-system-prompts/00-overview.md` and choose task, workflow, or multi-turn.
3. For input formatters, read `../../20-inputs/00-overview.md` and define the output contract.
4. Read `../../30-generation/01-context-engineering.md`.
5. Select 0–2 techniques or thinking frameworks only when they address a specific failure mode.
6. Record decisions in `steps/04-architecture.md`.

## Output

`steps/04-architecture.md` should explain:

- Chosen artifact type.
- Chosen prompt type, if applicable.
- Input contract.
- Techniques/frameworks selected or deliberately omitted.
- Tradeoffs and constraints.
